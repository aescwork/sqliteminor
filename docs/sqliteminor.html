add(members)[source]¶

Add (create) a row in the table whose name is assigned to self.table.

Args:
members (tuple): A tuple with the values to be inserted in the database. The values must be from left to right (first to last)

in the order in which columns of the target table were created.

For example if a table was created with a statement like:

CREATE TABLE trees(Id INT, tree_name TEXT, age_in_years INT)

and a row is to be inserted for a tree with an Id of 1, a tree_name of “ash”, and age_in_years of 200 then the values in the members tuple must also be in this order: (1, “ash”, 200).

table (str): The name of the table to which the row is to be added.

Returns:
Number of rows added or None if there was a problem executing the statement.

Warning

This method as written can only execute one INSERT at a time.

Note

If adding a row which contains a date, be sure to render the date in the YYYY-MM-DD format. Example: ‘2017-05-20’. The date can be either a string (corresponding to the TEXT data type in an sqlite table) or a datetime.date object (corresponding to the DATE data type in an sqlite table).

Retrieve a group of rows from a table whose data in a specified column (col) falls within a certain range. The range is assessed as being composed of values between and including a greater-than- and a less-than- reference value: ‘frm’ and ‘to’.

read_search_rows(target_col, search_val, search_specs=[True, False, False])[source]¶

Retrieve one or more rows whose specified column has the passed-in value.

Args:

target_col (str): The name of the column to be searched for the value in search_val. search_val (str) ...(other): The value to search for in the column specified by col. exact_match (boolean):

search_specs (list) -

[exact_match (boolean), search_val_at_start (boolean), search_val_at_end (boolean)] :

If exact_match is True, 1. have the query look for an exact match,
  1. search_val_at_start and search_val_at_end are disregarded.
  2. Can be used to find a number (as opposed to a string).

If search_val_at_start is True, Only return the row if the search_val is at the beginning of the target_col column specified by target_col. If search_val_at_end is True, Only look for the search_val if it is at the end of target_col column specified by target_col.

If both search_val_at_start and search_val_at_start are True, the search_val can be anywhere in the the target_col column. If both search_val_at_start and search_val_at_start are False, the search_val must match the entire column.

Returns:

rows (list): A list containing each row retrieved (rows contain all of their columns)
or None if there was a problem executing the statement.
