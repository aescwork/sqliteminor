sqliteminor: UsageÂ¶

  • Instantiate an Object:

    The sqliteminor object needs a connection (conn) object, which provides a connection to an sqlite database, and the name of a table in the sqlite database.

    The conn can be created directly or optionally with an sqlitemgr object.

    In Python, the sql statements used to work with an sqlite database table are executed with a cursor. Once the sqliteminor object has a conn, it creates its own cursor which it uses internally to run sql statements against a database.

    import sqlite3
    conn = sqlite3.connect("path/to/db/")
    sm = SQLiteMinor(conn, "my_table")
    or
    sm = SQLiteMinor()

  • Set or re-set the conn and table:
    sm.conn = conn
    sm.table = "my_new_table"

  • add() -- Add a Row of Data to a Database Table:

    Pass in a tuple with the data to be added.
    The tuple must have its members placed in it in the order in which the columns of the table are arranged, left to right. Or, the data should be placed in the tuple in the same order as the CREATE TABLE statement shown above:
    row_data = (19, "Apple", "deciduous", 20, '2017-07-14')
    sm.add( row_data)
    This method returns the number of rows added (this number should always be 1).

  • read_all(select_column="*") -- Get All Rows from the Table:

    The select_column argument specifies which columns to return with the row data that is retrieved. To specify particular column-data of returned rows, pass one or more names of the columns in a list. The default argument is an asterisk -- * -- which means return all the columns in the row.
    The search_scope argument can have one of two possible arguments: "has-any" or "has-all". has-any specifies that the search returns any rows which have any or all of the search values passed in (if more than one search value) with search_value. has-all specifies that only rows which have all of the search values passed in are returned.
    rows = sm.read_all()
    Returns a list with all rows in the table. Each row's data is grouped into a tuple. The list is empty if there are no rows in the database.
    print rows #prints the raw data in rows
    or
    for row in rows:
    print row[0], " -- ", row[1], " -- ", row[2], " -- ", row[3], " -- ", row[4]

  • read_range(frm, to, col, select_column="*") -- Retrieve a range of rows based on the contents of a specific row:

    The select_column argument specifies which columns to return with the row data that is retrieved. To specify particular column-data of returned rows, pass one or more names of the columns in a list. The default argument is an asterisk -- * -- which means return all the columns in the row.
    Example, get all the rows whose tree_id is between 2 and 10 (see chart above):
    rows = sm.read_range(2, 10, tree_id)
    Returns a list with the retrieved rows in the table. (This will also include rows with tree_id 2 and 10, if present). Each row's data is grouped into a tuple. The list is empty if there are no rows in the database.
    Returns None (<type 'NoneType'>) if there was a problem.
    print rows #prints the raw data in rows
    or
    for row in rows:
    print row[0], " -- ", row[1], " -- ", row[2], " -- ", row[3], " -- ", row[4]

  • read_search_rows(target_column, search_value, search_scope="has-any", select_column="*") -- Retrieve rows whose data in the specified column (target_column) contains a search value (search_value):

    The select_column argument specifies which columns to return with the row data that is retrieved. To specify particular column-data of returned rows, pass one or more names of the columns in a list. The default argument is an asterisk -- * -- which means return all the columns in the row.
    The search_scope argument can have one of two possible arguments: "has-any" or "has-all". has-any specifies that the search returns any rows which have any or all of the search values passed in (if more than one search value) with search_value. has-all specifies that only rows which have all of the search values passed in are returned.

    (The following examples use an asterisk -- * -- in the search_value to specify whether the argument passed in should only be at the beginning, anywhere, or at the end of the specified column of each row.)
    Example, get all the rows whose 'name' column has "el" in it (see chart above):
    1. rows = sm.read_search_rows("name", "el" ) #gets all rows whose data in the 'name' column matches "el" exactly
    (Should return)
    [ ] (empty)

    2. rows = sm.read_search_rows("name", "el*") # gets all rows whose data in the 'name' column matches "el" at the beginning
    # -- note asterisk at the end of the search_value argument.
    (Should return)
    [(5, u'Elm', u'deciduous', 100, datetime.date(2017, 2, 11)), (11, u'Elder', u'deciduous', 40, datetime.date(2017, 2, 11))]

    3. rows = sm.read_search_rows("name", "*el") #gets all rows whose data in the 'name' column matches "el" at the end --
    # note the location of the asterisk at the beginning of the argument passed in for the search_value.
    (Should return)
    [(15, u'Common Hazel', u'deciduous', 30, datetime.date(2017, 5, 2))]

    4. rows = sm.read_search_rows("name", "*el*") #gets all rows whose data in the 'name' column matches "el" anywhere.
    # -- note the asterisks on either side of the search_value argument.
    (Should return)
    [(5, u'Elm', u'deciduous', 100, datetime.date(2017, 2, 11)), (11, u'Elder', u'deciduous', 40, datetime.date(2017, 2, 11)), (15, u'Common Hazel', u'deciduous', 30, datetime.date(2017, 5, 2)), (18, u'Guelder Rose', u'deciduous', 900, datetime.date(2017, 2, 11))]

    Returns a list with the last n rows in the table.
    The list is empty if there are no rows in the database.

  • read_by_date_part(self, part, column, search_val, select_column="*") -- A method for specifically searching by dates. Retrieve one or more rows whose date has the passed-in day, month or year.

    The part argument This specifies which part of the date to search by. There can be one of three values: "day", "month" or "year".
    The column is the actual name of the column which contains the date.
    The search_val is the numerical value for the day (calendar date), month or year.
    The select_column argument specifies which columns to return with the row data that is retrieved. To specify particular column-data of returned rows, pass one or more names of the columns in a list. The default argument is an asterisk -- * -- which means return all the columns in the row.

    Example, get all the rows whose column is called 'sample_date', which has '02' (February) for its part, and whose whose search_val is 'month'. (see chart above):
    rows = read_by_date_part('month', 'sample_date', '02')
    (Should return row numbers)
    [5, 11, 12, 14, 16, 17, 18]

    Returns a list with the retrieved rows in the table. Each row's data is grouped into a tuple. The list is empty if there are no rows in the database.
    Returns None (<type 'NoneType'>) if there was a problem.
    print rows#prints the raw data in rows
    or
    for row in rows:
    print row[0], " -- ", row[1], " -- ", row[2], " -- ", row[3], " -- ", row[4]

  • update(self, target_column, new_val, reference_column, reference_column_value) -- Update one or more rows in a database table.

    This method uses asterisks as described above. See read_search_rows for complete details on how to use them in searching row data.

    Returns a number of how many rows were deleted.
    Returns None (<type 'NoneType'>) if there was a problem.

    Example, update all of the rows whose "tree_type" column matches exactly a value of "evergreen" to a value of "coniferous" (see chart above):
    number_rows_updated = sm.update( "tree_type", "coniferous", "tree_type", "evergreen")
    Example, update all of the rows whose "name" column has a substring "thorn" somewhere in its name to a value of "shrub" in its "tree_type" column:
    number_rows_updated = sm.update( "tree_type", "shrub", "name", "*thorn*") # note the asterisks on both sides of search_value.

  • delete(self, reference_column, ref_column_value, search_scope="has-any") -- Delete one or more rows in a database table.

    The search_scope argument can have one of two possible arguments: "has-any" or "has-all". has-any specifies that the search returns any rows which have any or all of the search values passed in (if more than one search value) with search_value. has-all specifies that only rows which have all of the search values passed in are returned.

    This method uses asterisks as described above. See read_search_rows for complete details on how to use them in searching row data.


    Returns a number of how many rows were deleted.
    Returns None (<type 'NoneType'>) if there was a problem.

    Example, delete all of the rows whose "tree_type" column has a value of "evergreen" (see chart above):
    number_rows_deleted = sm.delete( "tree_type", "evergreen") # number_rows_deleted should = 5.
    Example, delete all of the rows whose "name" column has a substring "thorn" somewhere in its name:
    number_rows_deleted = sm.delete( "name", "*thorn*") # number_rows_deleted = 3.

  • Get the results of the Method Call using the result and status members:

    Did the method call succeed or fail? Check the result:
    sm.result
    print sm.result # sm.result will either be 'OK' if successful or 'FAIL' if not.
    ... if sm.result is 'FAIL', any error message from a caught exception will be in sm.status:
    sm.status
    print sm.status # if 'FAIL', sm.status will contain a message about what happened.
